  • Week 5

    The proper was to write a literature review was introduced. The voice must be passive and cautious and must not go to deep into a single source, nor should it only skim upon various details of the topic. Phrasing is vitally important in a literature review, due to the fact that it might not convey the proper meaning that the author had intended. This could lead to a misunderstanding of those who are reading the review and may cause wrong ideas about the topic area to arise from this. The author must not have his opinion anywhere in the text, this is because an opinion inherently contains bias, and a literature review is simply to provide a document about a particular subject.

This would allow me to set various parameters whereby i could change the formatting, positioning, etc. of a specific piece of content on the page. This was coupled with the idea that it is bad practice to have form and content, i.e. HTML and css in a single HTML document. This is because it is much harder to maintain and add or remove content from a page if the styling is adding to the size and getting in the way. As well as this, having a separate CSS file will prevent procedural website generation, since that could look different on various devices. I was also introduced to the "link" tag which would allow me to link a HTML page to a CSS sheet in order for me to use the various form changing pieces written in the Style Sheet.
